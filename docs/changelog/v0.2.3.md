# v0.2.3 — Infrastructure: PHP proxy eliminates CORS dependency

**Date:** 2026-06-25

## Fixed

- Teacher login (and all future API calls) now work on the school network and mobile
  - Root cause: school network's SSL inspection proxy returned `400` for browser `OPTIONS`
    preflight requests; mobile networks timed out connecting to `vogel-api.duckdns.org` directly
  - Added `frontend/api/index.php` — a thin PHP reverse proxy on `klaasvogel.nl` that forwards
    all `/api/*` requests to the FastAPI backend server-to-server
  - Added `frontend/api/.htaccess` — Apache rewrite rule so `/api/teacher/login` routes to
    `index.php` without the `.php` extension
  - Updated `frontend/js/config.js` — `API_BASE` now points to `/natuurkunde/dobbelstenen/api`
    in production and `http://localhost:8000` in local dev

## How it works

Browser calls `https://www.klaasvogel.nl/api/...` (same origin — no CORS preflight).
The PHP script on `klaasvogel.nl` forwards the request to `vogel-api.duckdns.org` over a
server-to-server HTTPS connection (same path as a working curl call). The response, including
`Set-Cookie` headers, is passed back with the `Domain=` attribute stripped so session cookies
are bound to `klaasvogel.nl`.

## Notes

- FastAPI backend and its CORS config are unchanged — the proxy bypasses CORS entirely.
- VPN / internal-network HTTPS timeout is unresolved (hairpin NAT); not a production issue.
