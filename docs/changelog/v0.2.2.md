# v0.2.2 — Infrastructure: teacher HTTPS login fixed

**Date:** 2026-06-24

## Fixed

- Teacher login now works end-to-end from the browser via `https://vogel-api.duckdns.org`
  - Root cause: `vogel-api.duckdns.org` was absent from stunnel's SNI routing table; HTTPS
    connections silently fell through to the OpenVPN fallback and timed out after 20–30 s
  - Added `[nginx_api]` service to `stunnel.conf` with the Let's Encrypt cert for
    `vogel-api.duckdns.org` (npm-4 archive folder, mounted as `/etc/stunnel/certs/vogel-api`)
  - Used `connect = nginx:80` (not `:443`) — stunnel terminates TLS and must forward plain HTTP
    to NPM; connecting to port 443 caused the "plain HTTP request sent to HTTPS port" error
  - Removed SSL / Force SSL from the NPM proxy host for `vogel-api.duckdns.org` — NPM now
    handles the domain as plain HTTP (TLS is already terminated by stunnel upstream)

## Notes

- **stunnel.conf is baked into the Docker image** (COPY in `Dockerfile.stunnel`). Future config
  changes require `docker compose build stunnel && docker compose up -d stunnel`; a plain
  `docker restart stunnel` will silently use the old image.
- bcrypt cost 12 on NUC hardware takes ~3 s — expected and acceptable for a rare teacher login.
- **School network proxy blocks CORS preflights** — the school's SSL inspection proxy returns
  `400` for browser `OPTIONS` requests, and HTTPS to `vogel-api.duckdns.org` times out from
  mobile networks. Fixed in v0.2.3 via a PHP reverse proxy on `klaasvogel.nl`.
- VPN / internal-network access to `https://vogel-api.duckdns.org` still times out (hairpin
  NAT limitation). Workaround: test without VPN. Not a production issue.
