# v0.3.6 — Local dev/test environment + Docker quick-start

**Date:** 2026-07-02

## Fixed

### Backend
- Migration `b0ab4e9460bd_milestone_2_1_auth_and_schools` used `op.add_column` /
  `op.create_foreign_key` directly on the `classrooms` table, which SQLite can't apply
  outside of batch mode (`ALTER TABLE ... ADD CONSTRAINT` is unsupported there). This
  silently broke `alembic upgrade head` against a local SQLite dev database, leaving
  `dev.db` with an incomplete schema and every DB-touching endpoint returning 500. Wrapped
  the column/FK changes in `op.batch_alter_table(...)`, which works identically on
  MySQL/Postgres and now also works on SQLite.

## Added

### Dev tooling
- `scripts/bootstrap_test_classroom.py` — bootstraps a throwaway teacher, school, and
  classroom via the API (registration → admin approval → classroom creation) and prints a
  `join_code`, so the Milestone 3 student flow can be tested end-to-end without the
  Milestone 5 teacher dashboard UI existing yet. Works around a real gotcha: the session
  cookie is `Secure`, and unlike browsers, Python's `http.cookiejar` (used by `httpx`) has
  no `localhost`-is-secure exemption — it silently drops `Secure` cookies sent over plain
  `http://`. The script's `login()` helper re-inserts the cookie into the client's jar by
  hand after each login.
- `docker-compose.yml` — self-contained API + MySQL stack for anyone running their own
  instance, independent of the NUC's personal multi-service compose file.

### Documentation
- `docs/claude/local_test_environment.md` — full local dev/test walkthrough: `.env` setup
  for SQLite, running migrations, serving frontend + backend on matching `localhost`
  origins (and why that specifically matters for CORS and cookies), using the bootstrap
  script, and a troubleshooting table for the issues hit while setting this up.
- Rewrote `README.md` (previously a two-line placeholder): project description, tech
  stack, local dev quick-start, and Docker quick-start.
