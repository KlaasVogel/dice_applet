# v0.3.0 — Milestone 2.1: Multi-Teacher Auth & School Model

**Date:** 2026-06-29

## Added

### Backend
- `School`, `Teacher`, `TeacherSchool` DB models with Alembic migration
  (`b0ab4e9460bd_milestone_2_1_auth_and_schools`)
- `POST /auth/login` — unified login: no email → admin (`.env` hash); email present → DB teacher
- `POST /auth/register` — teacher self-registration with school request workflow
  - `@carmelhengelo.nl` domain enforced
  - New school: created inactive, status `pending_admin`
  - Join existing school: status `pending_school`
- `POST /auth/logout` — clears session cookie
- `GET /auth/me` — returns current session role and identity
- `GET /admin/requests` — lists pending teacher-school requests (admin only)
- `POST /admin/requests/{id}/approve` — approves request; activates school if `pending_admin`
- `POST /admin/requests/{id}/reject`
- `GET /admin/schools` — all schools (active + inactive)
- `GET /admin/teachers` — all teachers with their school links
- `GET /teacher/schools` — schools where caller is an approved teacher
- `GET /teacher/pending-requests` — `pending_school` requests for caller's schools
- `POST /teacher/pending-requests/{id}/approve` and `/reject`
- `POST /teacher/classrooms` — now requires `school_id`; verifies caller is approved for that school
- `GET /teacher/classrooms` — scoped to caller's approved schools
- 20 new pytest tests (SQLite in-memory, no MySQL required)

### Frontend
- Login modal: Docent / Admin pill tabs; email field shown only for teacher tab
- Admin tab: password-only, matches existing `.env`-hash admin flow
- Registration form view (`#view-register`): email, password, new-school vs. join-existing-school radio
- "Nog geen account? Verzoek indienen →" link from teacher tab

## Changed

- `POST /teacher/login` and `DELETE /teacher/logout` removed; replaced by `/auth/*`
- `require_teacher` dependency now returns `teacher_id: int` instead of `None`
- Test suite switched from MySQL lifespan to SQLite in-memory (`aiosqlite`)

## Fixed

- Frontend tab active state: changed from subtle underline to solid blue pill (clearly visible)
