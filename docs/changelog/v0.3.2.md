# v0.3.2 — Milestone 3 Phase 2: Dataset & Measurement Endpoints

**Date:** 2026-07-01

## Added

### Backend
- `GET /student/activities` — returns status of all 4 activity slots; lazily creates missing
  `StudentDataset` rows so the student always sees 4 tiles
- `GET /student/activities/{activity}` — returns full dataset detail + ordered measurements;
  also creates the dataset lazily if not yet started
- `PUT /student/activities/{activity}/measurements` — bulk-replaces all measurements for one
  player within an activity; 403 if dataset is locked, 422 if player is not 1 or 2
- `POST /student/activities/{activity}/request-unlock` — sets `unlock_requested=True`;
  400 if dataset is not locked
- `src/dice_applet/services/student_data.py` — service layer with `get_or_create_dataset`,
  `get_measurements`, `bulk_replace_measurements`
- `src/dice_applet/routers/student_data.py` — new router, registered under `/student` prefix
- New schemas: `MeasurementIn`, `MeasurementOut`, `MeasurementBulkRequest`, `ActivityStatus`,
  `DatasetDetail`
