# v0.4.0 — Milestone 4: Interactive Decay Graphs (p5.js)

**Date:** 2026-07-02

## Added

### Frontend
- `frontend/js/graph.js` — reusable `createDecayGraph(containerEl, options)` component:
  instance-mode p5.js, `noLoop()` + manual `redraw()` (static chart, not an animation),
  `{ destroy(), resize(), update(lines, opts) }` API
- Smooth curve rendering via monotone cubic Hermite interpolation (Steffen's method) →
  `bezierVertex()`, chosen over p5's built-in `curveVertex()`/Catmull-Rom after
  verifying the latter would visibly overshoot on a real wiggle in the Activity 3
  fixture data (`...,62,55,56,48,...`)
- Hover/touch crosshair: red vertical + horizontal line(s) snapped to the nearest data
  point, with a coordinate label; two simultaneous horizontal crosshairs on dual-line
  graphs
- `frontend/js/graph-mock-data.js` — fixture data for all four activities
- Activities 1 & 2: graph reflects live keystrokes from the on-screen measurement table
  (no network dependency for the graph itself — same input events already driving the
  existing debounced save)
- Activities 3 & 4: dual-line graph from mock data (both players' curves, legend with
  color swatches) — real paired data isn't available yet, see Milestone 4.1 below
- p5.js loaded via CDN in `frontend/index.html`; new `#activity-graph` card + legend
  markup inside the activity view
- New i18n keys: `graph_axis_x`/`graph_axis_y` (axis labels), plus `col_player1`/
  `col_player2` (legend labels — these were planned in Milestone 3 but never actually
  added; added now since the graph legend needed them)
- New CSS: `.graph-card`, `#activity-graph` sizing, `.graph-legend` swatches;
  `--color-accent` (`#f59e0b`) added to `:root` for the Player 2 line

### Roadmap
- New **Milestone 4.1 — Shared Activity Workspace** section in `ROADMAP.md`: two
  students will be able to share one dataset for Activities 3 & 4 (one creates it, the
  partner joins via personal code, both can toggle roles and see all data). Not
  implemented yet — the current data model has no link between paired students'
  datasets (`GET /student/activities/{activity}` only ever returns the requesting
  student's own rows), which is why M4's dual-line graph stays on mock data for now.

## Fixed

- Real bug caught during manual verification: a unit mismatch in the Bezier
  control-point calculation (`tangent * dx` instead of `tangent`, where the tangent was
  already computed per index-step, not per pixel) caused the curve to oscillate wildly
  between points instead of tracing a smooth line. Fixed in `graph.js`.

## How to view the test graphs yourself

1. Start the backend and frontend per `docs/claude/local_test_environment.md`
   (`uvicorn ... --reload` on port 8000, `python3 -m http.server 5500 --directory
   frontend` for the frontend).
2. Get a join code:
   ```bash
   .venv/bin/python3 scripts/bootstrap_test_classroom.py
   ```
   (admin password is whatever you set locally — see that doc if you don't remember it)
3. Open `http://localhost:5500/index.html`, join with the printed code.
4. **Activity 1 or 2** (single-line, live): open the tile, then type a few numbers into
   the "Dobbelstenen"/"Dice count" column — the graph above the table should redraw
   after each keystroke, always starting from `(0, 100)`.
5. **Activity 3 or 4** (dual-line, mock data): open the tile, pick either role — you'll
   see a fixed two-color curve (blue = Player 1, amber = Player 2) with a legend below
   it, regardless of which role you picked. This is fixture data, not live — see the
   Milestone 4.1 note above for why.
6. **Hover** your mouse anywhere over either graph: a red crosshair should appear,
   snapped to the nearest roll, with a coordinate label (two labels on the dual-line
   graph).
7. **Language toggle** (flag icons, top left) while an activity view is open: axis
   labels and the legend should switch NL/EN immediately.
