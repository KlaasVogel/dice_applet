# v0.3.5 — Milestone 3 Phase 5: Activity View

**Date:** 2026-07-01

## Added

### Frontend
- Full `#view-student-activity` layout: back button + title, role selector (activities 3 & 4),
  activity description card, lock banner, measurement table, add-row button, save indicator
- `openActivity(activity)` — shows role selector for activities 3 & 4; loads immediately for 1 & 2
- `_selectRole(player)` — stores player role and loads activity data
- `_loadAndRenderActivity()` — fetches `GET /student/activities/{activity}`, renders description,
  lock banner state, and table rows
- `_renderTable()` / `_appendRow()` — builds table rows with input listeners; minimum 12 rows
- `_onCellChange()` — auto-extend: adds a new row when a non-zero value is entered in the last
  row; stops when 0 is entered at row ≥ 12 (end-of-experiment signal)
- `_scheduleSave()` / `_save()` — 800 ms debounced `PUT /student/activities/{activity}/measurements`
  sending all filled rows for the current player
- `_showSaveIndicator()` — 2 s flash, green for success, red for failure
- `_requestUnlock()` — `POST /student/activities/{activity}/request-unlock`; disables button on success
- `_cancelPendingSave()` — clears debounce timer on back-navigation
- 14 new i18n keys (NL + EN): role selector, table headers (`col_roll`, `col_dice`), `add_row`,
  `saved`, `save_failed`, `locked_notice`, `request_unlock`, `unlock_requested`, and all 6
  activity description variants (`activity_1_desc`, `activity_2_desc`, `activity_3/4_desc_p1/p2`)
- New CSS: `.activity-nav`, `.btn-back`, `.role-buttons`, `.lock-banner`, `#measurement-table`,
  `.col-roll`, `.dice-input` (spin buttons hidden), `.btn-add-row`, `.save-indicator`
  (`.save-ok` / `.save-err`)
- 5 new event listeners in `app.js`: back button, role 1 & 2, request-unlock, add-row
